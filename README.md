# Программно-аппаратный комплекс (ПАК)
---
## Выполнил:
Фадеев Егор, Группа ИКС-432
---
Серверное приложение на C++, принимающее телеметрию с Android-устройства, сохраняющее данные в базу данных PostgreSQL и отображающее их в графическом интерфейсе с интерактивной картой и тепловыми картами покрытия сети.

## Зависимости:
- **OpenGL + GLEW** — рендеринг
- **ImGui + ImPlot** — графический интерфейс и графики
- **libzmq** — приём данных с Android по ZeroMQ
- **libpq (PostgreSQL)** — хранение телеметрии
- **libcurl** — загрузка тайлов карты OpenStreetMap
- **nlohmann/json** — парсинг JSON
- **stb_image / stb_image_write** — работа с PNG-изображениями

## Архитектура:
- **Андройд приложение `(Kotlin)`**: https://github.com/nevertoomuch/AndroidProject
- **Серверная часть `(C++)`**: https://github.com/nevertoomuch/backend-server

## Описание модулей серверной части:

- **main.cpp**:
Точка входа. Подключается к базе данных PostgreSQL, запускает поток загрузки тайлов карты и поток ZeroMQ-сервера (`RunServer`) в отдельных потоках, после чего запускает графический интерфейс. Обрабатывает сигналы `SIGINT` и `SIGTERM` для корректного завершения.
---
- **server.cpp / server.h**:
ZeroMQ REP-сервер, слушающий порт `7777`. Принимает JSON-пакеты с Android-устройства, извлекает GPS-координаты, данные сотовых сетей (LTE, GSM, NR) и уровень сигнала (RSRP). Обновляет глобальное хранилище `data_store`, сохраняет каждую запись в PostgreSQL через `SaveDataToDB` и ведёт лог последних 50 событий.
---
- **database.cpp / database.h**:
Работа с базой данных PostgreSQL через `libpq`. Реализует подключение (`ConnectToDatabase`), запись телеметрии (`SaveDataToDB`) и загрузку точек для карты (`LoadMapDataFromDB`). Структура `MapPoint` хранит координаты, RSRP, RSRQ, RSSI, высоту, PCI, EARFCN, тип сети и временну́ю метку. Имеется функция переподключения к БД при отказе работы самой БД.
---

- **data_structures.cpp / data_structures.h**:
Общие структуры данных и глобальные переменные. `Telemetry` хранит текущее состояние (координаты, тип сети, уровень сигнала). `SignalHistory` накапливает историю RSRP по каждому PCI за последние 200 точек для построения графиков в реальном времени. `OfflineData` содержит данные, загруженные из БД для отображения на карте.

---
- **gui.cpp / gui.h**:
Графический интерфейс на базе ImGui + ImPlot + SDL2 + OpenGL. Отображает текущие показатели телеметрии, интерактивную карту с тайлами OSM и точками измерений, график сигнала в реальном времени по PCI, панель управления тепловой картой и лог событий.
---
- **idw_heatmap.cpp / idw_heatmap.h**:
Генератор тепловых карт методом обратно взвешенных расстояний (IDW). Поддерживает критерии RSRP, RSRQ, RSSI и высоту, фильтрацию по PCI и EARFCN. Результат сохраняется в PNG-файл с цветовой кодировкой качества сигнала. Генерация выполняется в отдельном потоке с возможностью отмены и отслеживанием прогресса.
---
- **mercator.cpp / mercator.h**:
Утилиты для работы с проекцией Меркатора. Реализует преобразования между географическими координатами (lat/lon), координатами Меркатора и номерами тайлов OSM. Используется при отображении карты и наложении точек измерений.
---

- **tiles.cpp / tiles.h**:
Кэш тайлов карты OpenStreetMap. Загружает тайлы сначала с диска (папка `build/<zoom>/<x>/<y>.png`), при отсутствии — по HTTP через libcurl с серверов OSM, затем сохраняет на диск. Декодирует PNG через `stb_image` и хранит RGBA-данные для рендеринга через OpenGL. Загрузка выполняется асинхронно фоновым воркером.
